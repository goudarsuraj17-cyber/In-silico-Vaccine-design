# Contributing to Cryptococcus neoformans Vaccine Design Project

Thank you for your interest in contributing to this bioinformatics research project! This document provides guidelines for contributing.

## üéØ How Can I Contribute?

### Reporting Bugs

If you find bugs in the code or issues with the documentation:

1. Check the [Issues](https://github.com/yourusername/cryptococcus-vaccine-design/issues) to see if it has already been reported
2. If not, create a new issue with:
   - Clear, descriptive title
   - Detailed description of the problem
   - Steps to reproduce
   - Expected vs. actual behavior
   - Screenshots if applicable
   - Your environment (OS, browser, etc.)

### Suggesting Enhancements

We welcome suggestions for improvements:

1. Open an issue with the `enhancement` label
2. Clearly describe the enhancement
3. Explain why it would be useful
4. Provide examples if possible

### Code Contributions

1. **Fork the Repository**
   ```bash
   git clone https://github.com/yourusername/cryptococcus-vaccine-design.git
   cd cryptococcus-vaccine-design
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make Your Changes**
   - Write clear, commented code
   - Follow existing code style
   - Add documentation for new features
   - Test thoroughly

4. **Commit Your Changes**
   ```bash
   git add .
   git commit -m "Add: Brief description of changes"
   ```
   
   Use conventional commit messages:
   - `Add:` for new features
   - `Fix:` for bug fixes
   - `Update:` for updates to existing features
   - `Docs:` for documentation changes
   - `Style:` for formatting changes
   - `Refactor:` for code refactoring

5. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Open a Pull Request**
   - Provide a clear description of changes
   - Reference any related issues
   - Include screenshots for visual changes

## üìã Code Style Guidelines

### HTML
- Use semantic HTML5 elements
- Maintain proper indentation (2 spaces)
- Include ARIA labels for accessibility
- Keep IDs and classes descriptive

### CSS
- Follow BEM naming convention where applicable
- Group related properties together
- Comment complex selectors
- Use CSS variables for repeated values

### JavaScript
- Use ES6+ features
- Write modular, reusable functions
- Add JSDoc comments for functions
- Handle errors gracefully
- Avoid global variables

### Python (for analysis scripts)
- Follow PEP 8 style guide
- Use type hints
- Include docstrings
- Write unit tests for functions

## üî¨ Scientific Contributions

For bioinformatics contributions:

### Data Contributions
- Include data source and methodology
- Provide proper citations
- Follow FAIR principles (Findable, Accessible, Interoperable, Reusable)
- Document data formats

### Analysis Scripts
- Comment algorithm steps clearly
- Include parameter descriptions
- Provide example inputs/outputs
- Cite relevant publications

### Methodology Improvements
- Provide scientific justification
- Include references to literature
- Compare with existing methods
- Document validation results
